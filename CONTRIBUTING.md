Our Technical Workflow
To maintain high code quality (especially for university review), please follow these steps:
1. Fork the Repository: Create your own copy to work on.
2. Create a Feature Branch: git checkout -b feature/AmazingFeature.
3. Follow the FSM Pattern: Ensure all new logic fits within the existing RobotState structure.
4. Commit with Meaning: Use descriptive messages like feat: add GPS waypoint logic.
5. Open a Pull Request (PR): Describe your changes in detail for review.
